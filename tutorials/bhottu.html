Quoting

Quoting adds an entry to a database containing a name and a line.
Examples:

BOTNAME, quote <NAME> QUOTE_TEXT

Example:
Bottu, quote <example> This is an example

Replies

Replies are complicated, and thus need to be broken down into seperate parts.

Replies

Replies are one of the more useful features of bottu. Adding a reply allows ops and users to create
customized echoing commands. Examples and syntax below:

BOTNAME, trigger <reply> reply

Example:
BOTNAME, hey <reply> Hello!

Vars

Adding vars allows you to randomize or theme some replies. For example
adding "water", "juice" and "milk" to $DRINK will cause $DRINK to equal
any of those three words when called inside a reply. Examples below

Example:
BOTNAME, suggest a drink <reply> Why not try $DRINK ?

Adding to variables:

BOTNAME, add STRING to VARNAME

NOTE: If the variable does not currently exist, it is created when you add the first string

Echoing

Echo is the simplest command you can use with Bottu. Examples below:

Example:
BOTNAME, say TEXT This just causes Bottu to return TEXT.

Spewing

Name and concept shamelessly stolen from Trent of 7chan (Apologies!)
Using the spew command, Bottu will return 1 random line said by the given nick

Example:
BOTNAME, spew like NAME

Greeting

If you're an authuser (bottu will refer to you as a "GOD" if you are)
you can set bottu to greet someone upon entering the channel.

Example:
BOTNAME, greet NAME GREETING

NOTE: The bot will not allow people to select their own greetings.

BOTNAME, don't greet NAME
Will remove the greeting for selected nick.

Colors

If the bot detects a hexadecimal color code (eg, #EEFF00) in a message
it will return the name of that color (if it knows it). It will also
return a link to an image of that colour, so you can name it yourself

Example:
BOTNAME, add color #HEXCODE DEFINITION

Will add a description for the given color.
